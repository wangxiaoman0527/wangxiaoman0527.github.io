告别本地编译！用GitHub Actions让Hexo博客自动起飞 🚀

你是否也曾厌倦了每次更新博客都要在本地敲那重复的三件套？hexo clean && hexo g && hexo d 说多了都是泪啊！今天，就让我们解放双手，让博客自动部署，从此只需专注写作！

为什么要自动化？懒是第一生产力！😴

想象一下这样的场景：你在咖啡馆灵感迸发，用手机写好了一篇绝妙的文章，却因为电脑不在身边无法部署更新… 或者换了新电脑，又要重新配置一堆Node.js环境…

别再让技术限制你的创作！使用GitHub Actions，你只需要：写文章 → 推送代码 → 喝杯咖啡 ☕，剩下的交给机器人！

准备工作：给你的博客找个智能管家 🧙

1. 仓库分工明确

  • 源码仓库：存放你的Hexo所有家当（文章、主题、配置）
  • 页面仓库：专门存放生成的静态网页（比如 你的名字.github.io）

2. 密钥配对的魔法

想要让GitHub Actions帮你干活，得先给它一把钥匙：

  • 公钥放在服务器或GitHub Deploy Keys里（记得勾选写权限！）
  • 私钥藏在GitHub Secrets里，起个名字比如 SERVER_SSH_KEY

3. GitHub Token的通行证

去GitHub设置里生成一个Personal Access Token，需要repo权限，然后也存到Secrets里，取名 GH_TOKEN

测试一下你的自动化魔法 🔧

  1. 推送代码到main分支
  2. 去GitHub仓库的Actions页面围观机器人干活
  3. 如果失败，仔细阅读错误日志（机器人会告诉你哪里出了问题）
  4. 成功的话，就可以去你的博客刷新看看了！

进阶玩法：让自动化更智能 🧠

自动备份主题配置

有些主题修改后容易被覆盖，可以添加步骤自动备份你的自定义配置

图片自动压缩

在构建前自动压缩文章中的图片，提升加载速度

自动提交草稿

甚至可以设置自动将特定分支的草稿部署到测试站点

结语：享受自动化的美好时光 🎉

从现在开始，你可以在任何设备上写博客：平板、手机、甚至网吧电脑！只要能把Markdown文件推送到GitHub，剩下的就交给自动化流程吧。

你的时间很宝贵，应该花在创作上，而不是重复的部署命令上。
